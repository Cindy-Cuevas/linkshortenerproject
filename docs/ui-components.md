# UI Components Guidelines

## Overview

This project uses **shadcn/ui** exclusively for all UI components. Custom components should not be created when shadcn/ui provides an equivalent solution.

## Core Principles

1. **Always use shadcn/ui components** - Never create custom UI components from scratch
2. **Follow shadcn/ui patterns** - Use components exactly as documented in the shadcn/ui library
3. **Extend, don't replace** - If customization is needed, extend shadcn/ui components rather than building alternatives

## Component Installation

When a component is needed:

```bash
npx shadcn@latest add [component-name]
```

This will:
- Add the component to `components/ui/`
- Install any required dependencies
- Configure the component according to `components.json`

## Available Components

shadcn/ui provides components for:
- **Forms**: Input, Textarea, Select, Checkbox, Radio, Switch, Label, Form
- **Layout**: Card, Separator, Tabs, Accordion, Sheet, Dialog
- **Navigation**: Button, Dropdown Menu, Navigation Menu, Command
- **Feedback**: Alert, Toast, Badge, Progress, Skeleton
- **Data Display**: Table, Avatar, Calendar, Popover, Tooltip
- And many more...

Refer to [shadcn/ui documentation](https://ui.shadcn.com) for the complete list.

## Usage Patterns

### Basic Component Usage

```tsx
import { Button } from "@/components/ui/button"
import { Input } from "@/components/ui/input"

export function MyForm() {
  return (
    <form>
      <Input placeholder="Enter text" />
      <Button type="submit">Submit</Button>
    </form>
  )
}
```

### Styling with Tailwind

shadcn/ui components accept `className` prop for additional Tailwind styling:

```tsx
<Button className="w-full bg-primary hover:bg-primary/90">
  Full Width Button
</Button>
```

### Variants

Most shadcn/ui components support multiple variants:

```tsx
<Button variant="default">Default</Button>
<Button variant="destructive">Destructive</Button>
<Button variant="outline">Outline</Button>
<Button variant="ghost">Ghost</Button>
```

## Component Organization

- **UI Components**: `components/ui/` (auto-generated by shadcn)
- **Composed Components**: Create composition components that combine multiple shadcn/ui components
- **Page Components**: Use shadcn/ui components directly in pages/layouts

### Example: Composed Component

```tsx
// components/link-card.tsx
import { Card, CardHeader, CardTitle, CardContent } from "@/components/ui/card"
import { Button } from "@/components/ui/button"

export function LinkCard({ title, url }: { title: string; url: string }) {
  return (
    <Card>
      <CardHeader>
        <CardTitle>{title}</CardTitle>
      </CardHeader>
      <CardContent>
        <p>{url}</p>
        <Button variant="outline">Copy Link</Button>
      </CardContent>
    </Card>
  )
}
```

## Form Handling

Use shadcn/ui's Form components with React Hook Form:

```tsx
import { useForm } from "react-hook-form"
import { Form, FormControl, FormField, FormItem, FormLabel } from "@/components/ui/form"
import { Input } from "@/components/ui/input"
import { Button } from "@/components/ui/button"

export function MyForm() {
  const form = useForm()
  
  return (
    <Form {...form}>
      <FormField
        control={form.control}
        name="username"
        render={({ field }) => (
          <FormItem>
            <FormLabel>Username</FormLabel>
            <FormControl>
              <Input {...field} />
            </FormControl>
          </FormItem>
        )}
      />
      <Button type="submit">Submit</Button>
    </Form>
  )
}
```

## Important Notes

- **Do not modify** files in `components/ui/` directly - they are managed by shadcn/ui
- **Use the `cn()` utility** (from `lib/utils.ts`) for conditional class merging
- **Check shadcn/ui docs first** before implementing any UI element
- **Install only needed components** - don't install the entire library at once

## When to Use Client Components

shadcn/ui components often require `'use client'` directive because they use:
- Event handlers (onClick, onChange, etc.)
- React hooks (useState, useEffect, etc.)
- Browser APIs

```tsx
'use client'

import { Button } from "@/components/ui/button"
import { useState } from "react"

export function Counter() {
  const [count, setCount] = useState(0)
  
  return (
    <Button onClick={() => setCount(count + 1)}>
      Count: {count}
    </Button>
  )
}
```
